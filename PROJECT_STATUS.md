# ResilienceIQ — Project Status

A full-stack web app that gives workers a 0–850 **AI Career Resilience Score** — structured like a credit score — showing how exposed their role is to AI displacement.

---

## Stack

| Layer    | Tech                                                                 |
|----------|----------------------------------------------------------------------|
| Frontend | Next.js 14 (App Router), TypeScript, React, Tailwind CSS             |
| Backend  | Django 4.2, Django REST Framework, Python 3.11+                      |
| AI       | Google Gemini API (`gemini-2.0-flash`)                               |
| Database | SQLite (dev) · PostgreSQL-ready config in `settings.py`              |

---

## Features shipped

### Scoring engine ([backend/scoring/scoring_engine.py](resilienceiq/backend/scoring/scoring_engine.py))
- **0–850 weighted model** across 5 factors:
  1. Task Automation Resistance (max 200)
  2. Human Judgment Demand (max 170)
  3. Industry AI Adoption Speed (max 140, inverted — slower = higher)
  4. Sociotechnical Barriers (max 140)
  5. Skill Adaptability (max 200 — education + experience)
- **65+ hard-coded occupation profiles** with raw 0–10 scores per factor.
- **Fuzzy job-title matching** via `difflib.get_close_matches` (cutoff 0.80).
- **Gemini fallback**: if no profile matches, Gemini classifies the role into the same 4-factor schema.
- **Adaptability**: education level (max 160) + years of experience (max 40).

### REST API ([backend/scoring/views.py](resilienceiq/backend/scoring/views.py))
- `POST /api/score/` — returns score, label, 5 factors, and 3 recommendations.
- CORS-enabled; DRF configured with no auth required.

### Frontend flows
- **Landing page** ([frontend/app/page.tsx](resilienceiq/frontend/app/page.tsx)) — hero, static gauge preview, factor explainer.
- **Score form** ([frontend/app/score/page.tsx](resilienceiq/frontend/app/score/page.tsx)) — job title, industry, education, years.
- **Results page** ([frontend/app/score/results/page.tsx](resilienceiq/frontend/app/score/results/page.tsx)) — animated gauge, factor breakdown, action plan, What-If modeler.

### Share & model tools
- **Animated SVG gauge** ([frontend/components/ScoreGauge.tsx](resilienceiq/frontend/components/ScoreGauge.tsx)) with `requestAnimationFrame` easing + cleanup.
- **What-If modeler** ([frontend/components/WhatIfModeler.tsx](resilienceiq/frontend/components/WhatIfModeler.tsx)) — re-runs adaptability in pure client-side TS; no API call.
- **Shareable score card** ([frontend/lib/scoreCard.ts](resilienceiq/frontend/lib/scoreCard.ts)) — Canvas API generates 1200×630 PNG sized for LinkedIn/Open Graph.
- **Copy-to-clipboard** share text.

### Hardening
- Bug audit complete: font loader, arc geometry, Canvas `toBlob` async, fuzzy-match threshold, DRF auth bypass, Gemini model ID.
- Graceful fallback recommendations across 3 tiers when Gemini is unavailable or rate-limited.

## Where the data comes from

1. **Occupation profiles** — a hand-curated Python dict inside `scoring_engine.py` (`JOB_PROFILES`) with ~65 common roles. Each role has four 0–10 scores (`ar`, `jd`, `ia`, `sb`) informed by labor-economics research and public AI-exposure studies. **This is static data authored directly in code — no external labor-stats API is called.**
2. **Education / experience weights** — static Python constants (`EDUCATION_POINTS`, experience curve).
3. **Unknown job titles** — when fuzzy matching fails, Gemini is asked to estimate the same 4 factor scores for that role. This is the only case where the factor numbers themselves come from an LLM.
4. **Recommendations** — three upskilling actions per score are generated by Gemini, personalized to the user's job title, industry, and factor breakdown. A static 3-tier fallback list is used if Gemini fails.

## Where AI is used

**One model, two call sites, both in [scoring_engine.py](resilienceiq/backend/scoring/scoring_engine.py):**

| Call site                          | Purpose                                                              | Fallback if it fails              |
|------------------------------------|----------------------------------------------------------------------|-----------------------------------|
| `_get_ai_job_profile(...)`         | Score an unknown job title on the 4 factor dimensions                | Generic "unknown role" baseline   |
| `_get_gemini_recommendations(...)` | Generate 3 personalized upskilling recommendations                   | Static tier-based recommendations |

Model: **`gemini-2.0-flash`** via `google-generativeai==0.8.3`.
Key: read from `GEMINI_API_KEY` env var in [backend/.env](resilienceiq/backend/.env).

**AI is *not* used for**: computing the 0–850 score (deterministic math), education/experience weighting, the gauge UI, the What-If modeler, or the score card image — those are all pure code.
